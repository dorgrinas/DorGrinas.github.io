E-commerce Business Transaction Analysis - SQL/Oracle

This project focuses on exploring one year of sales data from a UK-based e-commerce store with 500K rows, utilizing SQL for in-depth analysis of transaction trends and customer behaviors. I took the dataset from Kaggle, a platform known for sharing datasets contributed by others.

    The dataset comprises the following columns:

  • TransactionNo (categorical): Unique six-digit number for each transaction, with "C" indicating cancellations.
  • Date (numeric): Date when each transaction was generated.
  • ProductNo (categorical): Unique five or six-digit character for product identification.
  • Product (categorical): Name of the product/item.
  • Price (numeric): Price per unit of the product in pound sterling (Â£).
  • Quantity (numeric): Quantity of each product per transaction; negative values indicate cancellations.
  • CustomerNo (categorical): Unique five-digit number for each customer.
  • Country (categorical): Name of the country where the customer resides.
